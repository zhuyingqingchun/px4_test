# 完整飞行链路检查清单

适用前提：你已经能稳定完成“起飞 + 悬停”。

接下来建议把验证目标从单点能力，推进到完整闭环能力。

## 1. 最小闭环目标

一次完整验证建议至少覆盖以下 10 步：

1. DDS / Agent 连通
2. PX4 ready
3. ROS 2 节点启动正常
4. 切入 Offboard
5. 解锁
6. 起飞到目标高度
7. 定点悬停保持
8. 小范围位移后回到原点
9. 降落并落地稳定
10. 上锁并完整退出

## 2. 通过判据

仅当以下项目同时满足，才建议判定为“完整闭环成功”：

- [ ] 未出现阻塞级报错
- [ ] Offboard 稳定进入
- [ ] 解锁成功
- [ ] 起飞高度达到阈值
- [ ] 悬停阶段无明显持续漂移
- [ ] 位移阶段轨迹连续，无明显跳变
- [ ] 返回原点后误差在可接受范围内
- [ ] 降落完成且落地稳定
- [ ] 退出流程无残留关键进程
- [ ] 第二次重复执行仍可成功

## 3. 推荐记录项

| 项目 | 建议记录 |
|---|---|
| 目标高度 | 例如 1.5m / 2.0m |
| 悬停时长 | 例如 10s / 30s |
| 位移方式 | 直线 / 方形 / 8 字 |
| 是否返航 | 是 / 否 |
| 是否降落 | 是 / 否 |
| 是否有红字 | 无 / 有 |
| 红字是否阻塞 | 是 / 否 |
| 本轮是否成功 | 是 / 否 |

## 4. 失败时优先判断的层级

1. **控制层**：Offboard 是否真的进入，是否真的发送 setpoint。
2. **连接层**：Agent、ROS 2 topic、QGC、Gazebo 是否存在链路中断。
3. **时序层**：启动顺序是否变化，是否存在首次成功、二次失败。
4. **环境层**：残留进程、端口占用、旧日志误导。

## 5. 最小建议

在完成 3~5 轮重复启停验证之前，不建议大改控制逻辑。
先把“完整闭环流程”跑稳，再进入轨迹扩展和状态机改造。
