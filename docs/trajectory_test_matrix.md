# 轨迹测试矩阵

适用前提：起飞与悬停已经稳定成功。

目标：把“会飞起来”推进到“能按计划飞”。

## 1. 轨迹测试顺序

建议严格按由简到繁的顺序推进：

1. 直线前进再返回
2. 小方形轨迹
3. 不同高度切换
4. 多航点停留
5. 8 字轨迹

不要同时改多个变量。每轮最好只改一个核心量。

## 2. 推荐测试矩阵

| 轮次 | 轨迹类型 | 高度(m) | 速度级别 | 航点数 | 每点停留(s) | 是否成功 | 备注 |
|---|---|---:|---|---:|---:|---|---|
| 1 | 直线往返 | 1.5 | 低 | 2 | 2 |  |  |
| 2 | 小方形 | 1.5 | 低 | 4 | 2 |  |  |
| 3 | 高度切换 | 1.5→2.0 | 低 | 3 | 2 |  |  |
| 4 | 多航点停留 | 2.0 | 中 | 5 | 3 |  |  |
| 5 | 8 字轨迹 | 2.0 | 中 | 自定义 | 0 |  |  |

## 3. 重点观测项

每轮至少观察以下内容：

- [ ] 轨迹开始前 Offboard 是否稳定
- [ ] 起飞后是否达到预期高度
- [ ] 轨迹是否连续，无明显抖动或跳点
- [ ] 是否存在明显超调
- [ ] 是否存在持续漂移
- [ ] 是否能在结束后回到安全状态
- [ ] 是否能正常降落与退出
